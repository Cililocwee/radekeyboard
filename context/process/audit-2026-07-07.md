# Codebase Audit — Rade Keyboard (2026-07-07)

A snapshot of the app's state at the time the GoManzanas agentic tooling was ported
in, plus the issues found and what was done about them. Keep this for future
reference — especially the **standing code-smell warnings** at the bottom, which are
still true after the fixes.

## What the app is

Native Android IME (custom soft keyboard) for Vietnamese/Rade, written in Java.
Single Gradle module, ~1,700 lines of Java across 6 files. AGP 8.4, minSdk 21,
compile/target SDK 35. No backend, no database, no network.

- `ModernInputMethodService` — IME: key handling, tone marks, auto-caps, spacing
- `ModernKeyboardView` — ~1,000-line custom `Canvas` keyboard view
- `VietnameseText` — pure tone-mark/combining helper (added during this audit)
- `MainActivity` / `OnboardingActivity` / `KeyboardSetupChecker` — setup flow

## Build health at audit time

`./gradlew assembleDebug` succeeded cleanly (SDK 35 installed locally). The entire
current app lived on branch `enhance/better-ui-v1`, ~30 commits ahead of `main` with
0 behind — `main` was badly stale. This audit's work was bundled into a PR to catch
`main` up.

## Standing code-smell warnings (still true — read before extending)

These were **not** fully resolved and remain risks for future work:

- **`ModernKeyboardView` is a ~1,000-line monolith** doing layout, drawing, touch,
  popups, and timers in one class. It's the single most likely place to introduce
  bugs. Prefer extracting pure helpers (as was done for `VietnameseText`) over adding
  more to it. A real refactor (separating layout math, rendering, gesture/timer state,
  and the popup) is still owed.
- **Tone-mark / combining logic is subtle.** Order is base-char-first, marks apply
  only to vowels, and the space/punctuation paths interact with auto-capitalization.
  It now has unit-test coverage via `VietnameseText`, but the *service-level*
  orchestration (delete-and-recommit, auto-caps timing) is still only manually
  verifiable — treat changes there carefully.
- **No landscape handling and a fixed 48dp key height.** Dark mode is addressed;
  responsive/landscape layout is not.
- **No instrumented test coverage.** The IME behavior end-to-end (enable → type →
  tone marks → onboarding) is verified only by hand.
- **Signing is not wired into Gradle.** Release builds are currently unsigned. Play
  uploads require the one-time keystore + `signingConfig` setup documented in
  `docs/deploy-to-google-play.md` / `/prepare-to-deploy`.
